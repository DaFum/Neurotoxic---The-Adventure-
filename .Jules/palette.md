## 2024-05-15 - [ARIA Labels and Focus States]
**Learning:** Some custom icon-only buttons (`X` to close dialogue) are missing `aria-label`s, which makes them inaccessible to screen readers. In addition, interactive lists like the inventory use elements that act as toggle buttons but lack `aria-pressed` states and `focus-visible` styling for keyboard navigation.
**Action:** Always verify that buttons with only icons include descriptive `aria-label`s. Ensure elements functioning as toggles have `aria-pressed` and `focus-visible` styles to help users navigate correctly using keyboards.

## 2024-03-31 - Add ARIA pressed states and focus-visible rings to floating toggles
**Learning:** Floating interface toggles (like HUD visibility and Pause buttons) in WebGL/Canvas apps often miss critical accessibility attributes because they sit outside typical semantic layouts. They require explicit `aria-pressed` attributes to communicate their toggle nature to screen readers, and robust `focus-visible` styling is necessary since they often blend into dark backgrounds or lack native focus outlines. When using `aria-pressed` to denote a toggle state, the `aria-label` should remain static (e.g., "Toggle pause menu" rather than switching between "Pause" and "Resume") so the screen reader can accurately announce the state as "[Label], toggle button, pressed/not pressed".
**Action:** Always verify that interactive elements functioning as toggles include static `aria-label`s, `aria-pressed` states, and `focus-visible` styling (e.g. `focus:outline-none focus-visible:ring-2`) for proper keyboard navigation and accessibility compliance.
