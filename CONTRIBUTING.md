# Mitwirken (Contributing)

Wir freuen uns über Beiträge zu **NEUROTOXIC: Grind The Void**! Bitte folge diesen Schritten, wenn du Änderungen vorschlagen möchtest.

## Entwicklungsumgebung einrichten

1. **Repository klonen**

   ```bash
   git clone https://github.com/DaFum/Neurotoxic---The-Adventure-
   cd Neurotoxic---The-Adventure-
   ```

1. **Abhängigkeiten installieren**

   ```bash
   pnpm install
   ```

1. **Entwicklungsserver starten**

   ```bash
   pnpm run dev
   ```

## Workflow & Befehle

Bevor du einen Pull Request (PR) erstellst, stelle sicher, dass alle Tests und Qualitätsprüfungen erfolgreich durchlaufen.

- **Linting ausführen**

  ```bash
  pnpm run lint
  ```

- **Formatierung prüfen/anwenden**

  ```bash
  pnpm run format
  ```

- **Tests ausführen**

  ```bash
  pnpm run test
  ```

## Pull Request Guidelines

1. Erstelle einen Branch für dein Feature oder deinen Bugfix (`git checkout -b feature/mein-neues-feature`).
1. Nimm deine Änderungen vor und schreibe klare, verständliche Commit-Nachrichten.
1. Achte darauf, dass alle Checks (`lint`, `format`, `test`) bestanden sind.
1. Öffne den Pull Request und beschreibe detailliert:
   - Was wurde geändert?
   - Warum wurde es geändert?
   - Welche Issues werden dadurch gelöst (falls zutreffend)?
1. Warte auf ein Review durch einen Maintainer.
